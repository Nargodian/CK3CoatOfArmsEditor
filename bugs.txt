# Non-Critical Bugs Tracking

## Transform Widget Frame Handling - RESOLVED
- **Issue**: Transform widget doesn't account for different frames when aligning
- **Solution**: Added frame transform methods to canvas_widget for coordinate conversion
- **Implementation**: 
  - `get_frame_transform()` - returns current frame's scale and offset
  - `coa_to_frame_space()` - converts CoA coordinates (0-1) to frame-adjusted visual space
  - `frame_to_coa_space()` - converts frame visual space back to CoA coordinates
  - Updated `update_transform_widget_for_layer()` to send frame-adjusted coordinates to widget
  - Updated `_on_transform_changed()` to convert back to CoA space when applying transforms
- **Impact**: Transform widget now accurately shows and manipulates layer positions accounting for each frame's unique scale (0.85-1.0) and offset transforms
- **Status**: Fixed - tested with various frames
